# KC NRVV Follow-Ups Validation

## Scope

Implement NRVV follow-up issues #65 through #69:

- Align Need trace status with validation evidence.
- Add parse-friendly NRVV examples.
- Add NRVV optional/warning/strict profile behavior.
- Add `kc issue-record --nrvv-file`.
- Trace Requirements through plan items and verification evidence.

## Verification

- `npm.cmd run check`: passed.
- `npm.cmd test`: passed, 50 tests.
- `npm.cmd run pack:dry`: passed.
- `node lib/cli/index.js check --workspace . --mode pr ...`: passed.
- `dist/action/index.js`: regenerated by the build step.

## Behavioral Validation

- Need trace now becomes `addressed` only when validation status is passed and validation evidence is present.
- Explicit `verification_evidence[].requirement_refs` are preferred over broad inference when generating Requirement trace.
- `nrvv_profile: warning` emits NRVV findings without breaking default compatibility.
- `nrvv_profile: strict` makes missing or incomplete NRVV blocking.
- `kc issue-record --nrvv-file` loads structured NRVV YAML and rejects non-object YAML.
- Issue template and README files now show parser-friendly Requirement and Verification examples.

## Remaining Limits

- KC still treats NRVV as lightweight issue discipline, not a full requirements-management or ALM system.
- Reviewer usefulness of the new trace fields should continue to be validated through dogfooding.
